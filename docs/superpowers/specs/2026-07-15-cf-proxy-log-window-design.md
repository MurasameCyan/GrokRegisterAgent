# CF 独立代理日志输出窗口设计

## 背景

设置页已有「CF 独立代理」面板，用于配置并启动 `cfwp` 本地 HTTP/SOCKS 代理。后端 `cfwpManager` 已把 `cfwp` 的 stdout/stderr 写入 `/data/cfwp/<port>.log`，但前端目前只显示运行状态、pid、lastError 和本地代理地址，用户无法直接看到详细启动/退出/报错日志。

## 目标

在「CF 独立代理」面板底部嵌入一个日志输出窗口，让用户在保存、同步、启动、停止后可以直接查看最近 `cfwp` 日志，方便排查域名、token、端口、二进制、Docker/Linux 环境和网络问题。

## 非目标

- 不做实时 WebSocket tail。
- 不做历史日志文件切换。
- 不删除服务器日志文件。
- 不改动普通代理/代理池逻辑。
- 不引入新的大型日志系统。

## 用户界面

采用方案 A：底部嵌入日志窗口。

位置：放在现有「将使用本地代理」状态卡片之后、SSO/Auth 代理开关之前。

窗口包含：

- 标题：`cfwp 日志`
- 右侧显示日志路径，例如 `/data/cfwp/30000.log`
- 日志正文使用等宽字体，保留换行，最大高度滚动
- 空内容提示：`暂无日志，保存或启动 CF 独立代理后刷新。`
- 读取失败时在窗口内显示错误，不阻断设置页其它功能

按钮：

- `刷新`：重新读取日志
- `复制`：复制当前日志内容
- `清空显示`：仅清空前端当前显示内容，不删除服务器日志文件

## 后端接口

新增接口：

```http
GET /api/cf-proxy/log?tail=200
```

返回结构：

```ts
interface CfProxyLogResult {
  ok: boolean;
  logPath: string | null;
  content: string;
  truncated: boolean;
  error?: string;
}
```

接口行为：

- 从 `cfwpManager` 获取当前 `logPath`。
- 若没有日志路径或文件不存在，返回空内容和可读提示。
- 默认返回最近 200 行。
- 限制最大读取字节数，避免大日志卡死。
- 如果当前配置里有 token，返回前做脱敏：把 token 替换为 `******`。

## 数据流

1. 前端进入设置页并启用 CF 独立代理面板。
2. 前端调用 `getCfProxyLog()` 读取最近日志。
3. 用户点击同步/启动/停止：
   - 先走现有 CF 操作接口；
   - 更新状态；
   - 再刷新日志窗口。
4. 用户点击刷新时只读取日志，不改变进程状态。
5. 用户点击清空显示时仅清空 React state。

## 错误处理

- 后端读取失败：返回 `ok: false` 和 `error`，前端显示在日志窗口中。
- 日志文件不存在：返回 `ok: true`、空 content、`logPath`，前端显示空提示。
- token 脱敏失败不应导致接口失败；最多返回未脱敏以外的安全内容。实现时应确保只在 token 非空时替换。

## 测试与验证

- TypeScript 类型检查通过。
- 构建通过。
- 手动验证：
  1. CF 面板开启时可以看到日志窗口；
  2. 无日志时显示空提示；
  3. 启动/同步/停止后自动刷新日志；
  4. 复制按钮复制当前内容；
  5. 清空显示不删除服务器日志；
  6. 返回内容中不出现明文 token。
