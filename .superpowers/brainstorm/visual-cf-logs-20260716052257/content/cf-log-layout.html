CF 独立代理日志窗口：布局选择

目标是在现有 CF 独立代理面板里，让用户能直接看到 cfwp 启动/退出/报错输出，便于排查域名、token、端口、二进制和网络问题。

方案 A：面板底部嵌入日志（推荐）
Cloudflare Socks5/HTTP 本地代理
状态、同步、启动、停止
现有参数表单区域
cfwp 日志/data/cfwp/30000.log 
[05:22:11] cfwp start client_ip=:30000
[05:22:12] connected cf domain ...
[05:22:14] local socks/http ready
方案 B：折叠日志抽屉
现有 CF 参数表单
▶ cfwp 日志点击展开
A

底部嵌入日志窗口

最直接，用户保存/启动后不用找位置；适合当前面板已经是一个独立配置块。可设最大高度、自动滚到底、支持刷新/清空。

B

折叠日志抽屉

默认保持页面短；需要排查时展开。缺点是报错不如 A 醒目，用户可能忽略。

C

独立弹窗/侧边面板

日志空间最大，适合长日志；但实现和使用路径更重，不太符合这个小范围改动。

建议默认能力

  • 显示最近 N 行日志
  • 按钮：刷新、复制、清空显示
  • 启动/同步/停止后自动刷新
  • 不暴露 token（必要时后端脱敏）

先不做

  • 实时 WebSocket tail
  • 日志搜索和下载
  • 复杂过滤器
  • 多历史文件切换
